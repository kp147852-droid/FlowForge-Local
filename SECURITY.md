# Security Policy

## Reporting vulnerabilities
Please report security issues privately to the maintainer via GitHub Security Advisories or direct contact.

Include:
- affected component
- reproduction steps
- impact assessment
- remediation suggestion (if available)

## Security-sensitive scope
- file system automation actions (copy/move/rename/delete)
- webhook delivery and outbound integrations
- API authentication/session controls
- deployment configuration and secrets handling

## Disclosure practice
Please avoid public disclosure until a patch is available.
